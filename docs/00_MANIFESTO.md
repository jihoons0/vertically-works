Vertically Works

A Manifesto for Vertical Interfaces

⸻

Why This Exists

For decades, digital interfaces have been designed around a single assumption:

People read horizontally.

This assumption is embedded everywhere.

Operating systems.

Browsers.

Applications.

Design systems.

Component libraries.

Interaction patterns.

Animation.

Navigation.

Scrolling.

Even the language of interface design assumes left-to-right thinking.

Yet billions of people have a cultural history of reading vertically.

Chinese.

Japanese.

Korean.

For centuries, books, poems, newspapers, official documents, calligraphy, and literature were designed around vertical reading.

When computing arrived, these traditions adapted to horizontal software instead of software adapting to them.

The result is that vertical writing survives digitally as typography—not as interaction.

Vertically Works exists to challenge that assumption.

⸻

The Question

This project does not ask:

“Can we rotate text?”

It asks:

“If digital interfaces had been invented around vertical writing systems, what would software look like today?”

That question affects everything.

Navigation.

Motion.

Hierarchy.

Selection.

Scrolling.

Animation.

Components.

Artificial intelligence.

The answer is not simply rotating existing interfaces.

It requires rethinking interaction itself.

⸻

Our Belief

Writing direction is not visual decoration.

Writing direction is interaction.

Reading changes how people scan.

Scanning changes hierarchy.

Hierarchy changes navigation.

Navigation changes interaction.

Interaction shapes software.

Typography is only one layer.

Interaction is the real problem.

⸻

The Mission

Build the world’s most complete reference for vertical interface design.

Not through opinion.

Through research.

Experimentation.

Documentation.

Interactive examples.

Open discussion.

This project exists to make future designers ask better questions.

⸻

Principles

Interaction First

The purpose of this project is not to create beautiful mockups.

It is to improve interaction.

Every design decision should begin with user behavior.

Never aesthetics.

⸻

Respect Reading

Reading creates expectation.

Interfaces should reinforce those expectations rather than oppose them.

Motion.

Navigation.

Hierarchy.

Layout.

Spacing.

Interaction.

Everything begins with reading.

⸻

Teach Through Interaction

The best explanation is an experience.

Whenever possible, replace paragraphs with interactive demonstrations.

People remember what they explore.

Not what they read.

⸻

Show Reasoning

Never present solutions without explaining why.

The reasoning behind a component is often more valuable than the component itself.

Document tradeoffs.

Document alternatives.

Document uncertainty.

⸻

Research Is Never Finished

Many questions remain unanswered.

Where should drawers originate?

How should chat interfaces behave?

How should drag-and-drop work?

How should AI conversations flow?

Do not pretend certainty where none exists.

Leave room for discovery.

⸻

Accessibility Is Fundamental

Accessibility is not a feature.

It is part of interaction.

Every component should consider:

Keyboard navigation.

Touch.

Screen readers.

Reduced motion.

Contrast.

Mixed writing systems.

Accessibility should influence design from the beginning.

⸻

Simplicity Is Respect

Documentation should never compete with ideas.

The interface exists to reveal knowledge.

Not to impress.

Avoid decoration.

Avoid unnecessary animation.

Avoid complexity that does not improve understanding.

⸻

The Website Is Not The Product

The website is documentation.

The knowledge is the product.

If the website disappeared tomorrow, the ideas should survive.

Future outputs may include:

Applications.

Open source libraries.

Research.

Books.

Design systems.

Educational material.

AI tooling.

The website is simply the first expression of that knowledge.

⸻

Open Knowledge

Vertically Works is intended to become a shared reference.

Good ideas should spread.

Better ideas should replace them.

Research should remain transparent.

Contributions should improve understanding rather than establish ownership.

⸻

Looking Forward

Today’s software was shaped by assumptions made decades ago.

Many of those assumptions were reasonable.

Some deserve to be questioned.

Vertically Works exists to ask those questions.

Not because vertical interfaces should replace horizontal ones.

But because interaction design becomes stronger when it acknowledges that there has never been only one way to read.

⸻

One Sentence

Vertically Works explores how software changes when writing direction becomes the foundation of interaction rather than a typographic afterthought.
